# Decisions (ADRs)

Record architectural decisions here using short, numbered ADRs.

Template:

```
# ADR N: Title

Date: YYYY-MM-DD
Status: Proposed | Accepted | Superseded by ADR M

Context
- Why change is needed

Decision
- What is chosen and why

Consequences
- Positive/negative outcomes
- Operational impacts

References
- Links to tickets, PRs, docs
```
